4주차 실습 정리

오늘의 목표:
원형 / AABB / OBB Bounding Box 시각화
세 방식의 충돌 판정 차이 관찰

관찰:

1.원형 vs AABB
중심을 기준으로 거리만 닿으면 되는 원형과 달리 AABB는 직사각형의 범위로 판정하다보니 원형이었으면 닿지 않았을 경우도 닿는 경우가 있었다.
AABB가 원형보다 부자연스러운 판정이 더 자주 발생했다. 대신 한편으로는 박스끼리의 판정을 할때는 원형은 모서리쪽에 판정이 없어 AABB가 나은 경우도 있었다.

2.AABB vs OBB
AABB의 문제점은 물체를 회전시킬 때 자주 발생했다. OBB는 물체의 회전을 따라가는 직사각형이다 회전해도 적절한 크기를 유지했지만,
AABB는 회전하면 원래 크기보다 더 큰 직사각형이 되어 충돌하지 않았을 상황에서도 충돌이 나는 경우가 많았다. 심지어 AABB가 원형보다 그나마 뛰어난 점인
박스끼리의 판정에서도 AABB의 잘못된 크기 범위때문에 OBB보다 불안정했다.

3.AI와의 대화에서 배운 것:
편의성이나 더 좋은 성능을 사용한다는 것은 그만큼 잃는 것이 있다는 것을 알게 되었다. 예를 들어 sqrt 함수를 사용하는 것은 그냥 계산하는 것보다 연산비용이 크지만
그만큼 편의성이 좋기에 사용되는 것이고, AABB는 OBB보다 부정확한 판정을 내지만 OBB의 코드 구현이 복잡하고 연산값이 많기에 OBB의 장점이 불필요할 때는 대신 사용되는 것을 보며
알게 되었다.

4.내 게임에 적용한다면
만약 플랫포머를 만든다면 OBB가 가장 좋고 정확한 판정이겠지만 그만큼 구현이 복잡해지기 때문에
크게 타협을 본다면 AABB가 적당할 것이라 생각한다. 예를 들어 마리오 같은 오브젝트에 회전이 적게 발생하는 게임이라면
OBB의 필요성이 줄어들고 AABB로 대체할 수 있을 것이라 생각한다. 만약 회전이 필요하다 해도 AABB와 OBB를 혼합해서 사용할 것 같다.
