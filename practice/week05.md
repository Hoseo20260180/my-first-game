#5주차 실습 기록 

## 선택한 템플릿 및 주제 
Snake.py를 선택했으며 이를 변형해 끝까지 생존하여 클리어 시간으로 점수를 따지는 것이 아닌 특정 길이가 되기 전까지 최선의 전략으로 가장 많은 점수를 버는 것을 목표로 바꾸었다.

## 사용한 AI 프롬프트 요약
1. 코드 구조 설명 요청
 기존 스네이크 코드의 나누어져 있는 함수들의 기능을 파악하고자 AI에게 설명을 요청함.
-> 각 함수들이 각각 열매생성, 게임오버, 뱀 구현, 레벨 선택 등으로 이루어져 있음을 알게 됨.

2. 기획서 구체화 요청
게임에 추가할 기능(아이템, 속도 변화, 특수 효과 등)에 대한 아이디어를 얻을 수 있었고 애매한 게임의 목표를 바꿀 아이디어를 제공해줌.

3. 구현 가능성 확인
입력 지연 개선, 독 과일 추가, 점수 기반 속도 증가, 화면 색 반전 효과 등이 가능한지 물어봄
-> 전부 구현 가능한 기능 들이며 추가로 독과일의 추가 디버프나 색반전의 정확한 효과를 제시해줌.

##  AI 답변에서 도움이 된 것
기존 스네이크 코드의 각 함수들을 이해하는데 도움이 됨.
입력 처리 지연 문제를 개선하여 입력 처리와 이동 로직을 분리하여 반응 속도를 개선하는 방법을 알게 됨.
독 과일, 점수 기반 속도 증가, 화면 색 반전 효과 등 게임의 재미를 높이는 기능을 구체적으로 구현할 수 있었다.
단순 기능 추가뿐만 아니라, 게임 기획 측면(목표, 아이템 시스템, 난이도 조절 등)에 대한 아이디어도 얻을 수 있었다.

## AI 답변을 수정하거나 버린 것
일부 기능은 게임 난이도가 지나치게 높아질 수 있어 적용하지 않았음. (예: 과도한 속도 증가, 복잡한 아이템 시스템)
코드 구현 방식 중 일부는 현재 코드 구조에 맞게 간단하게 수정하여 적용함.
성능에 영향을 줄 수 있는 과도한 색반전 효과는 필요에 따라 제한하도록 함.
